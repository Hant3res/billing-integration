## Дефект №1: Ошибка подключения к MSSQL из-за отсутствия базы данных

| Поле | Значение |
|------|----------|
| **Описание** | Сервисы падают с ошибкой `Cannot open database "billing_db" requested by the login` |
| **Шаги воспроизведения** | Запустить `docker-compose up -d`. Сервисы не могут подключиться к БД |
| **Ожидаемый результат** | Сервисы создают таблицы автоматически |
| **Фактический результат** | SQLAlchemy не может создать таблицы, так как БД не существует |
| **Причина** | MSSQL требует явного создания базы данных перед созданием таблиц |
| **Решение** | Добавлен скрипт `init_mssql.py` для автоматического создания БД и таблиц |
| **Коммит** | `git commit -m "Step 6: Add Python script for MSSQL initialization"` |
| **Статус** | ✅ Исправлен |

## Дефект №2: Ошибка в Dockerfile (apt-key устарел)

| Поле | Значение |
|------|----------|
| **Описание** | Сборка Dockerfile падает с ошибкой `apt-key: not found` |
| **Шаги воспроизведения** | Запустить `docker-compose build`. Сборка падает на установке ODBC |
| **Ожидаемый результат** | Успешная установка msodbcsql18 |
| **Фактический результат** | Ошибка, так как apt-key удалён из Debian 12 |
| **Причина** | Microsoft инструкции не обновлены для новых версий Debian |
| **Решение** | Заменён apt-key на `gpg --dearmor` с сохранением ключа в `/usr/share/keyrings/` |
| **Коммит** | `git commit -m "Step 6: Fix Dockerfiles for Debian 12, remove apt-key"` |
| **Статус** | ✅ Исправлен |

## Дефект №3: Отсутствует компенсация (Saga) при ошибке оплаты

| Поле | Значение |
|------|----------|
| **Описание** | При ошибке оплаты счёт остаётся в статусе `pending`, остатки не возвращаются |
| **Шаги воспроизведения** | Вызвать checkout с тарифом, у которого нет остатков, или симулировать ошибку платежа |
| **Ожидаемый результат** | Счёт отменяется, остатки возвращаются |
| **Фактический результат** | Счёт остаётся активным, остатки не восстанавливаются |
| **Причина** | Отсутствует вызов компенсации в блоке catch при ошибке оплаты |
| **Решение** | Добавлены функции `compensate_invoice()` и `compensate_reservation()` в оркестратор |
| **Коммит** | `git commit -m "Step 7: Add stock check, Saga pattern with compensation"` |
| **Статус** | ✅ Исправлен |
