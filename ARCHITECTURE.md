# Архитектура интеграционного решения

## Общее описание

Разработана **микросервисная система выставления счетов (Billing System)** для автоматизации процессов:
- Управление тарифами и услугами
- Создание и выставление счетов
- Обработка платежей
- Оркестрация сквозного сценария оформления заказа

## Архитектурный стиль

- **REST API** — синхронное взаимодействие между модулями
- **Микросервисная архитектура** — каждый модуль независим
- **Saga Pattern** — для обеспечения транзакционности (компенсация при ошибках)

## Компоненты системы

| Компонент | Порт | Технологии | Описание |
|-----------|------|------------|----------|
| Tariff Service | 5001 | Python/Flask, MSSQL, Redis | Управление тарифами, проверка остатков |
| Invoice Service | 5002 | Python/Flask, MSSQL | Создание и управление счетами |
| Payment Service | 5003 | Python/Flask, MSSQL | Обработка платежей |
| Orchestrator | 5000 | Python/Flask | Оркестрация сценария checkout |
| MSSQL | 1433 | SQL Server 2022 | Хранилище данных всех сервисов |
| Redis | 6379 | Redis 7 | Кэширование тарифов |

## Схема взаимодействия

Клиент (Браузер/Frontend)
│
▼
Orchestrator (порт 5000) ──────────────► Tariff Service (порт 5001)
│ │
│ ▼
│ Redis (кэш)
│
├──────────────────────────────────► Invoice Service (порт 5002)
│
└──────────────────────────────────► Payment Service (порт 5003)
│
▼
MSSQL (общая БД)

## Сценарии работы

### Успешный сценарий оформления заказа
1. Проверка остатков тарифов
2. Резервирование тарифов
3. Создание счета
4. Обработка платежа
5. Возврат успешного ответа клиенту

### Сценарий с ошибкой (Saga)
1. Ошибка на любом шаге
2. Компенсация: отмена счета, возврат остатков
3. Возврат ошибки клиенту
